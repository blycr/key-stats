## Highlights

- **Smarter Top-Keys SQL** — raised the aggregation limit from 50 to 200 rows so that numpad and main-row digits (which get collapsed into the same display name) no longer get truncated before the merge step.
- **Reduced GC pressure** — `getActiveWindowTitle()` now reuses a `sync.Pool` buffer instead of allocating a new `[]uint16` on every keystroke.
- **Faster frontend rendering** — eliminated O(N²) progress-bar calculation and replaced linear key-cap lookups with a `Map`, cutting CPU work during each 500 ms poll.
- **Leaner CSS transitions** — removed `box-shadow` from the global `*` transition to cut unnecessary compositing overhead during rapid stat updates.

## Changes

- `internal/stats/stats.go`: `LIMIT 50` → `LIMIT 200` in the Top Keys aggregation query
- `internal/service/keyboard.go`: added `sync.Pool` for UTF-16 window-title buffers
- `frontend/src/App.svelte`: pre-compute `maxKeyCount` reactively instead of inside the `{#each}` loop
- `frontend/src/components/KeyboardMap.svelte`: build a `Map` for O(1) key-name → count lookups
- `frontend/src/app.css`: dropped `box-shadow` from the global theme-transition rule
- Version bump to 1.1.3

## Build

Download `key-stats.exe` and run. Data is stored at `%APPDATA%/key-stats/`.
