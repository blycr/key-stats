## Highlights

- **Date range query fix** — "Last 7 Days" and "Last 30 Days" now correctly aggregate across the full range (SQL `BETWEEN`) instead of returning only a single day's data
- **App breakdown** — new "Top Apps" card in the sidebar shows which applications receive the most keystrokes
- **UI polish** — wider title bar drag area (72→84px), no more browser right-click menu on the webview, favicon 404 eliminated

## Changes

- Added `GetDateRangeSummary` and exported `GetDateRangeStats` for frontend range queries
- Added `GROUP BY app_name` SQL query to populate `AppBreakdown` in stats summaries
- Fixed VK name mappings: Left/Right Win keys now distinguished (`LWin` / `RWin`), VK 223 mapped to `~`
- Replaced `GetForegroundWindow` with `FindWindowW("wailsWindow")` in `StartDrag` for reliability
- Removed demo fallback data from production frontend code
- Wired up `on:click` handler in SettingsPanel to close dropdowns when clicking elsewhere in the panel
- Marked `wailsjs/` auto-generated files as LF-only in `.gitattributes` to prevent CRLF churn on Windows
- Version bump to 1.1.1

## Build

Download `key-stats.exe` and run. Data is stored at `%APPDATA%/key-stats/`.
